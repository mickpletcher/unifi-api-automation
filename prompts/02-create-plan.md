Create or update an implementation plan for a completed spec.

Steps:
1. Read `.github/specs/<feature-slug>/spec.md`.
2. Create or update `.github/specs/<feature-slug>/plan.md`.
3. Use `.github/specs/templates/plan-template.md` as the base.
4. Define design decisions and file by file changes.
5. Include validation steps with exact commands.
6. Include rollout and backout steps.

Output:
1. Show implementation order.
2. Show files to edit.
3. Show validation commands.
